§ 01 — Goal

Convert today's P&L workflow into an agent-driven flow that compresses cycle time without weakening controls.

Three goals, all achievable simultaneously if the agent layer is designed around behaviours rather than tools, and humans stay accountable at every toll gate.

→

Fast-track

Agents run rule-based stages without queueing. Trader-lane reconciles, AVP-lane adjusts and finalizes — in parallel, not in series.

✓

Optimize

Anomaly triage, P&L explain, and commentary draft auto-generate. Controllers review exceptions instead of producing them.

◷

On-time

Predictable EOD delivery. Schedulers replaced by event-driven agents that respond to book-open events from MOTIF in real time.

Peter Warlock principle — if you can't describe an agent in one sentence, it is too complex.

Propose, not post

Agents draft adjustments and reclassifications. Humans authorize the GL write. No exceptions, no overrides, no "trusted" agents.

Drill-back at every gate

PC clicks any number → sees which agent produced it, with which prompt, calling which tools, returning what data. Full provenance.

Risk-based oversight

Above-threshold variance auto-routes to senior controller for review; sub-threshold auto-resolves with full audit trail.

§ 08 — Considerations

Key constraints for design

What Anthropic's design must account for, given the Barclays operating context.

MRM compliance

Any agent influencing a number on the GL or regulatory report falls under SR 11-7 and PRA SS1/23. Validation pack, drift monitoring, override-rate tracking are required from day one.

Determinism for numbers

LLMs reason; deterministic tools compute. Same inputs must produce same numbers at T+1. Snapshot the prompt and tool outputs as the audit record; the LLM narrative sits on top of the numbers, not in their place.

Workflow variation

FI Credit asset class had FOBO removed in June '25; reclassification still runs in workflow. The agent layer must accommodate variants without forking the codebase.

Master book vs name PNL

Granularity drives orchestration: ~200 name PNLs to monitor. One orchestrator instance per name PNL with shared toolset and isolated state.

Audit trail

Every prompt, tool call, and agent decision goes to immutable storage with hash-chain anchoring. WORM-grade retrievability for MRM and Internal Audit.

Reconciliation to baseline

Agent-produced P&L must reconcile to the manual baseline before any cut-over. Run in shadow mode for a full quarter alongside the existing process.

§ 09 — Open questions

What we want Anthropic's view on

Before we lock the design.

1
How do we validate the agent layer for MRM (SR 11-7 / PRA equivalent) when LLMs introduce non-determinism?
2
Risk-based human oversight: does PC review every step or only above threshold? What does the regulator expect?
3
Fan-out pattern for ~200 name PNLs — one orchestrator instance per PNL, or shared with state isolation?
4
Failure modes: when an agent errors mid-flow, what is the rollback / human handoff pattern?
5
Learning loop: how do we capture controller corrections to improve agents over time without retraining?
6
Recommended build sequence: which agent ships first, and which is the last to be granted any write capability?
