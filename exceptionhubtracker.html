Critical Review Points
🔴 FileWatcher→Hub handoff: if DB write succeeds but Hub fails to pick up, exception is silently lost. Verify retry or queue mechanism.
🔴 All stage transitions must be inside a single @Transactional block. Partial writes in a custom FSM corrupt state permanently.
🔴 Concurrent users on same exception: custom FSM has no built-in locking. Verify @Version optimistic locking on Exception entity.
🟡 All transition guard checks (role, prior stage, data completeness) must happen server-side, not only in UI button visibility.
🟡 FileWatcher must persist processed-file manifest to Oracle DB — in-memory state is lost on restart causing duplicate loads.
🟡 CLOSED state must produce a complete WORKFLOW_AUDIT row. Without this table there is zero regulatory traceability.
