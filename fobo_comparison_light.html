Recommendation — given infra uncertainty
Start with Quartz throttle in Orchestration — zero new infrastructure, ships fast, stops the burst at source. If RabbitMQ can be provisioned, layer it in to fix the consumption side with prefetchCount backpressure and built-in DLQ. Skip Kafka unless FOBO evolves into a platform-wide finance messaging bus.
Phase 1 — Now
Quartz + pool tuning
Rate-limit Orchestration dispatch. Increase Pool 2/4 queue size within 4 GB cap.
Phase 2 — If RabbitMQ approved
