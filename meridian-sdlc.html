Objective 01
Market & Stakeholder Discovery
Interview Bank CTOs, CROs, and Compliance leads. Identify top 5 pain points with current LLM sprawl. Map competitor landscape (Kong, LiteLLM, TrueFoundry).
  • 10+ CTO/CIO stakeholder interviews
  • Regulatory pain point mapping (SR 11-7, SOX, DORA)
  • Competitor feature gap analysis
  • Pricing model research (per-token vs. flat SaaS)
Objective 02
Architecture Blueprint & ADRs
Produce the Architecture Decision Records for every major technical choice — language, framework, data plane, control plane separation, and deployment model.
  • ADR-001: Java Spring Boot vs .NET Core for gateway core
  • ADR-002: Reactive (WebFlux) vs thread-per-request
  • ADR-003: Vector DB selection (pgvector vs Pinecone)
  • ADR-004: On-prem vs cloud-native deployment topology
  • ADR-005: Multi-tenant data isolation strategy
Objective 03
Product Spec & PRD
Write the Product Requirements Document covering MVP scope, feature prioritisation (MoSCoW), non-functional requirements (latency, throughput, availability), and SLA targets.
  • MVP feature list (MoSCoW prioritised)
  • NFR: <5ms gateway overhead, 99.99% uptime
  • Multi-tenant isolation requirements
  • Compliance checklist (SOX, SR 11-7, DORA)

Testing Strategy

  • Integration tests against mock LLM providers

Architecture Patterns
Key Design Decisions Executed
  • Hexagonal architecture — ports & adapters for all LLM providers
  • Filter chain pattern for control planes (ordered, composable)
  • Event sourcing for audit log (append-only, Kafka-backed)
  • Sidecar pattern for PII redaction (Presidio as separate container)
  • CQRS for admin portal (command vs read model separation)
  • Circuit breaker (Resilience4j) per provider endpoint

⚠️
Critical Phase: No shortcuts here. Every bank CTO will ask "did it pass pen test?" before signing. This phase is the product's license to operate in regulated environments.
Security Track
Penetration Testing & Threat Model
  • STRIDE threat modelling for all gateway surfaces
  • External pen test (CREST-certified vendor)
  • Prompt injection attack simulation
  • API fuzzing (OWASP ZAP + custom LLM payloads)
  • Dependency vulnerability scan (Snyk / OWASP Dep-Check)
  • Secrets scanning — zero secrets in codebase
  • All Critical/High findings must be remediated before Phase 4
Regulatory Track
Compliance Certification Workstreams
  • SR 11-7 — Model inventory, validation docs, ongoing monitoring plan
  • SOX — Audit trail certification: hash integrity verified, retention proven
  • DORA — ICT resilience testing: failover <30s, RTO/RPO documented
  • GDPR/CCPA — DPA templates, data residency controls, right-to-erasure tested
  • Basel III — Operational risk taxonomy mapped to AI failure modes
  • External legal review of data processing agreements
Resilience Track
Chaos Engineering & DR Validation
  • Chaos Monkey: random provider failure injection
  • Network partition simulation (split-brain gateway)
  • Provider outage: auto-failover verified <500ms
  • Disaster recovery drill: full restore from backup
  • Backup/restore RTO target: <4 hours
  • Multi-region active-passive topology validated
